文档使用说明

搜索

在搜索过程中应用的规则包括：

  • 可以使用引号进行多词短语的精确搜索（例如，“big array”只有在这两个词连续且与在搜索框中输入的内容完全一致时才返回结果）。这种类型的搜索称为短语搜索。
  • 空格字符分隔关键词（例如，big array 被视为两个单独的关键词）。
  • 布尔搜索支持使用以下运算符：and（与），or（或），not（非）。当两个相邻的搜索词没有运算符时，默认使用 or 作为搜索运算符（例如，big array 等同于 big or array）。
  • 由冒号（“:”）、减号（“-”）、下划线（“_”）或点号（“.”）组合而成的词被视为一个单词。
  • 出现多个搜索关键词时，例如在搜索栏输入“A B C”时，这三个关键词在搜索时是“或”关系。
  • 每一个搜索结果右下方显示的星型符号，反映了搜索关键词在该结果中的出现次数。星型符号数量与搜索关键词命中次数成正比。

代码样例

DolphinDB 的函数页面、插件使用说明、API 接口说明以及教程提供了大量的代码样例以便用户在学习产品使用的过程中参考。参考这些代码样例中，需要注意以下几点：

  • 配置项。尽管不同的配置项在教程中或某些产品说明页面中因上下文的行文逻辑和内容主旨，使用了不同于默认配置的设定，在实际的生产环境中，请根据实际的需求做出相应的配置调整。
  • 文件路径。代码样例中使用的文件路径仅能反映编者在内容设计时使用的实际环境，与用户的实际部署环境或生产环境中的文件路径并不一定完全吻合。因此，在参考代码样例时，请用户务必根据实际部署环境中的文件目录做出相应的调整。
  • 名称。代码样例中的数据库名、数据表名、列名等名称通常为自定义的。用户在参考代码样例的过程中应根据实际的场景，进行相应的调整。
  • 环境清理。在某些涉及建库建表的例子脚本中使用了 dropDatabase 函数清除数据库，例如：
    dbPath="dfs://db_value_date"
if s.existsDatabase(dbPath):
    s.dropDatabase(dbPath)
    这并非要求或鼓励用户在实际环境中随意删除数据库，而是由于为了相似的例子脚本能够顺利运行，需要清除同名的数据库。
    重要： 在任何涉及到实际生产数据或重要数据的环境中，执行任何库表的删除操作前请务必谨慎，以免造成不必要的数据损失和业务影响。

文档版本

用户点击页面右上方灰色箭头，进入 DolphinDB 版本切换的下拉菜单。选择版本查看需要注意以下几点：

  • 版本查看。DolphinDB 使用<主要>.<中间>.<次要>.<补丁>来标识版本（例如，2.00.11.2），四个数字的依次增大表示版本的更新。您也可以在下拉选择菜单中直接选择最新版查看。
  • 版本变化。点击左侧目录中的版本说明，在 Server 的下拉列表中选择版本号查看相应版本更新历史，其中包含对应的版本下载、新功能、功能改进和缺陷修复等内容：
    • 版本下载中明确了版本、兼容等级和版本号。
    • 新功能和功能改进标注了更新内容的对应版本号。
    • 下拉列表从下到上为版本依次更新。
  • 版本兼容。
    • 点击版本兼容性说明查看 DolphinDB 版本兼容性等级标准。
    • 点击版本兼容性说明的下拉列表，选择查看每一级兼容性标准。
    • 对照兼容等级，查看对应版本兼容性标准的具体要求和升级指南，从而了解新版本对用户使用、系统兼容性的影响。
